후크

AI 코딩 에이전트가 한 대의 머신이 아니라 스스로 프로비저닝한 워커 네트워크에서 실행된다면 어떨까요? 대규모 작업을 분할해 배포하고, 진행 상황을 체크포인트로 저장해 크래시가 발생해도 작업이 유실되지 않도록 한다면요. 이것이 바로 HelixCode입니다. Go로 작성된 엔터프라이즈급 분산 AI 개발 플랫폼으로, SSH 기반 워커 풀, 자동 체크포인팅, 롤백, 그리고 멀티 프로바이더 LLM 통합 기능을 제공합니다.

왜 매력적인가

대부분의 “AI 코딩 에이전트”는 단일 프로세스가 단일 모델과 통신하는 구조입니다. 반면 HelixCode는 실제 개발 작업이 분할 가능하고 분산 처리될 수 있다는 아이디어에 기반해 설계되었습니다. 지능형 작업 분할과 의존성 관리를 수행한 후, 자동 체크포인팅을 통해 장기 실행 작업이 중단되어도 안전하게 보존됩니다. 이 플랫폼은 REST API, CLI, 터미널 UI, WebSocket이라는 네 가지 클라이언트 인터페이스를 제공하며, Model Context Protocol(MCP)을 다중 전송 방식으로 구현해 에이전트 툴링 생태계에 원활하게 통합됩니다.

또한 HelixCode는 이례적으로 광범위한 크로스 플랫폼을 지원합니다. Linux, macOS, Windows는 물론 Aurora OS와 SymphonyOS까지 지원하며, iOS와 Android용 프레임워크도 준비 중입니다. 저장소는 Go를 핵심으로 Shell, Kotlin, Swift, 웹 에셋 등으로 구성되어 있어 어디서든 플랫폼을 제어할 수 있는 구조를 갖추고 있습니다.

어려운 문제들

핵심 과제는 취약성 없는 분산 처리입니다. 코딩 작업을 원격 워커로 분할하는 순간, 분산 시스템의 모든 어려운 문제가 따라옵니다. 실제 의존성 경계를 따라 작업을 어떻게 나눌지, 어떤 워커가 어떤 부분을 담당하는지 추적할지, 워커가 작업 중 사라졌을 때 어떻게 복구할지, 결과를 어떻게 일관성 있게 병합할지가 모두 해결해야 할 과제입니다.

워커 관리는 또 다른 문제입니다. HelixCode는 사전 구성된 클러스터를 가정하지 않습니다. 대신 SSH 워커 풀과 자동 설치 기능을 제공해 원격 호스트에 접속해 자체적으로 설치를 완료하고, 워커를 등록하며, SSH를 통해 보안 상태를 모니터링해야 합니다. 이 부트스트랩 과정을 이질적인 머신에서 안정적으로 수행하는 것은 생각보다 훨씬 어렵습니다.

그다음은 LLM 계층입니다. 하나의 인터페이스 뒤에서 여러 프로바이더를 지원하고, 하드웨어(CPU/GPU/메모리)를 감지해 모델을 지능적으로 선택하며, 고급 추론 기법(사고 연쇄, 사고 트리)을 적용하려면 에이전트는 무엇을 생성할지만이 아니라 어디서, 무엇을 사용해 실행할지를 판단해야 합니다.

게임 체인저인 이유

작업 보존과 롤백 기능의 조합이 차별화 요소입니다. 중단된 지점부터 정확히 재개하고, 잘못된 변경을 롤백할 수 있는 AI 에이전트는 장기 자동화 작업을 도박이 아닌 운영 가능한 프로세스로 바꿔줍니다. HelixCode는 이를 계획, 빌드, 테스트, 리팩터링이라는 전체 개발 워크플로 모드와 결합해 데이터베이스에 프로젝트를 영구 저장하고, 멀티 세션 컨텍스트를 추적합니다. 단순한 “코드 생성”이 아니라 개발 라이프사이클 전체를 지원하는 플랫폼입니다.

MCP 구현도 중요합니다. 다중 전송 방식으로 Model Context Protocol을 지원함으로써 HelixCode는 에이전트 툴링 생태계의 소비자이자 참여자로서 역할을 수행할 수 있으며, Slack, Discord, 이메일, Telegram 등 다양한 채널로 알림을 전송해 분산 실행 중에도 인간이 실제로 모니터링하는 곳에 진행 상황을 보고할 수 있습니다.

가장 어려운 부분을 해결한 방법

시스템을 명확한 내부 모듈로 구성했습니다. 인증 및 세션 관리, 워커 풀 관리, 작업 관리 및 체크포인팅, 프로젝트 및 워크플로 관리, LLM 프로바이더 통합 등 각 모듈이 독립된 패키지로 존재합니다. 이러한 분리는 분산 동작을 다루기 쉽게 만듭니다. 작업 분할과 체크포인팅은 어떤 LLM 프로바이더나 전송 방식과 무관하게 독립적으로 고려할 수 있는 문제로 남습니다.

워커 네트워크의 경우, SSH 워커 풀이 스스로 설치되도록 구축했습니다. 운영자가 모든 노드를 사전에 준비할 필요 없이, 플랫폼이 SSH로 접속해 필요한 소프트웨어를 설치하고 워커를 등록한 후 지속적으로 상태를 모니터링합니다. 상태 모니터링 결과는 작업 관리에 반영되어 노드 성능이 저하되면 작업을 재할당할 수 있습니다.

복원력을 확보하기 위해 체크포인팅을 핵심 메커니즘으로 삼았습니다. 작업 관리 시스템은 진행 상황에 따라 체크포인트를 기록해 중단된 작업이 처음부터 다시 시작하지 않고 마지막 정상 상태에서 재개되며, 롤백을 통해 잘못된 단계를 되돌릴 수 있습니다. 이것이 장기 분산 자동화 작업을 안전하게 시도할 수 있게 하는 기반입니다.

모델 계층의 경우, 모든 프로바이더를 통합 인터페이스 뒤로 숨기고 하드웨어 감지 기능을 추가해 에이전트가 실행 중인 머신에 맞는 모델을 선택하도록 했습니다. 이 인터페이스 위에 도구 호출과 사고 연쇄/사고 트리 추론 기법을 적용해 추론 전략을 개선하더라도 프로바이더의 기본 구조를 건드릴 필요가 없습니다. 영속성 계층은 PostgreSQL(선택적으로 Redis)을 사용해 프로젝트, 세션, 워크플로 상태를 전체 분산 환경에서 지속적으로 관리합니다. 이는 단순한 데모가 아닌 실제로 운영 가능한 플랫폼의 차이를 만듭니다.
